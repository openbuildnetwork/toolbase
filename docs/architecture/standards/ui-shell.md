# UI Shell Standard

## Goal
Provide a consistent interaction model across all tools.

## Required Structure
1. Tool sidebar (left).
2. Context header (top).
3. Full-width content area in main panel.
4. Content grouped in section cards.

## Required States
1. Loading state.
2. Empty state.
3. Error state.
4. Success/valid state (if applicable).

## Controls
1. Primary action right-aligned in action bars.
2. Format/type selectors grouped near relevant actions.
3. Validation issues displayed inline as visible lists.

## Accessibility Baseline
1. Keyboard reachable action controls.
2. Explicit labels on inputs/selects.
3. Contrast-safe status colors.
