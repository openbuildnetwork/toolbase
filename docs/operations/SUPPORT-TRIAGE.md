# Support and Triage

## Issue Intake
Classify each issue by:
1. Tool
2. Severity
3. Reproducibility
4. Privacy/security impact

## Severity Levels
1. `P0` Product unusable or privacy/security risk.
2. `P1` Major workflow broken in one or more tools.
3. `P2` Degraded UX or partial feature break.
4. `P3` Cosmetic or low-priority improvement.

## Triage Workflow
1. Confirm reproduction steps.
2. Assign ownership.
3. Add regression test requirement.
4. Link affected docs and architecture decision if needed.
