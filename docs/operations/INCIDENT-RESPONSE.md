# Incident Response

## Scope
Incidents include outages, major regressions, or privacy-impacting defects.

## Immediate Response
1. Confirm incident severity.
2. Stop new risky releases.
3. Identify blast radius (tools affected).
4. Publish internal status update.

## Recovery Workflow
1. Mitigate impact quickly (rollback or hotfix).
2. Validate fix in impacted tools.
3. Monitor for recurrence.

## Postmortem
1. Timeline of incident.
2. Root cause.
3. Corrective actions.
4. Preventive changes (tests, guards, docs, ADRs).
